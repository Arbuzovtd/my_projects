<context>
Команда передает вам утвержденный Product Requirements Document (PRD) алгоритмического торгового робота (TradingView + Bybit + Telegram WebApp).
Ваша задача как Архитектора — перевести этот PRD в конкретный, пошаговый технический план реализации (Architecture & Implementation Plan), который затем будет передан Junior/Middle разработчикам (или другим ИИ-агентам) для написания кода.
</context>

<instructions>
Ваша задача — проанализировать предоставленный PRD и выдать исчерпывающий архитектурный документ. Вы должны продумать систему так, чтобы она была надежной, быстрой и легко масштабируемой, но без излишнего «оверинжиниринга» (overengineering).

Выполните следующие шаги:

Анализ: Внимательно прочитайте PRD и выделите ключевые технические риски (например, race conditions при чтении/записи config.json, обработка ошибок API).

Структура проекта: Разработайте файловую структуру проекта.

API Контракты: Опишите формат JSON-вебхуков, которые мы ожидаем от TradingView.

Управление состоянием: Предложите безопасный механизм работы с config.json в асинхронной среде.

План разработки: Разбейте реализацию на конкретные, изолированные задачи (спринты).

</instructions>

<constraints>

Технологии строго по PRD: Python 3.12+, FastAPI, aiogram 3.x, pybit (V5), Docker, uv.

Никаких баз данных: Используем только config.json, как описано в PRD. Не предлагайте Redis, PostgreSQL или SQLite.

Асинхронность: Все I/O операции (чтение файла, запросы к Bybit, Telegram API) должны быть строго асинхронными.

Не пишите готовую бизнес-логику целиком. Ваша цель — спроектировать систему (интерфейсы, структуры, файлы), а не написать весь код.
</constraints>

Пошаговый план реализации (Action Plan): Конкретный список задач для агента-кодера, начиная с настройки uv и заканчивая деплоем.
