# Инструкция по деплою торгового бота на сервер Timeweb Cloud (Франкфурт) с HTTPS

Эта инструкция описывает процесс с нуля: от покупки сервера до запуска бота через Docker Hub с автоматическим SSL-сертификатом (через Caddy).

---

## ШАГ 1: Покупка сервера и домена

1. **Регистрация / Вход:** Зайдите в панель управления [Timeweb Cloud](https://timeweb.cloud/).
2. **Покупка домена:**
   - Перейдите в раздел **"Домены"** -> **"Зарегистрировать домен"**.
   - Выберите любой дешевый домен (например, в зонах `.site`, `.space`, `.fun` - они обычно стоят 150-200 руб/год).
   - Завершите покупку. Домен нам нужен для подключения безопасного HTTPS (требование Telegram WebApp и TradingView).
3. **Создание сервера (VDS):**
   - Перейдите в раздел **"Облачные серверы"** -> **"Создать"**.
   - **Операционная система:** `Ubuntu 24.04` (или 22.04).
   - **Регион:** `Франкфурт` (Европа). Это обеспечит минимальный пинг до основных бирж.
   - **Конфигурация:** Выберите минимальный или базовый тариф (1 Core, 1 GB RAM, NVMe диск) — для бота этого более чем достаточно.
   - **Настройки:** Убедитесь, что стоит галочка "Публичный IP".
   - Нажмите **"Заказать"**. Дождитесь создания сервера (1-2 минуты). Пароль от `root` придет вам на почту и появится в панели.

## ШАГ 2: Привязка домена к серверу

Чтобы Caddy мог автоматически выпустить SSL-сертификат, домен должен указывать на IP-адрес вашего нового сервера.

1. Зайдите в раздел **"Домены"** в панели Timeweb.
2. Нажмите на шестеренку рядом с вашим купленным доменом -> **"Настройки DNS"**.
3. Добавьте/измените **A-запись**:
   - Хост (или Имя): `@` (что означает сам домен).
   - Значение (или IP-адрес): Вставьте `IP-адрес вашего нового сервера` из Франкфурта.
   - TTL: можно оставить по умолчанию.
4. *Важно: Обновление DNS может занять от 15 минут до нескольких часов, прежде чем домен начнет "открывать" ваш сервер.*

## ШАГ 3: Подготовка на вашем компьютере (Локально)

Перед тем как идти на сервер, вам нужно собрать актуальную версию бота и отправить её в Docker Hub.

1. Откройте терминал в папке с вашим проектом.
2. Соберите новый образ (замените `timurarbuzov` на ваш логин в Docker Hub, если он отличается):
   ```bash
   docker build -t timurarbuzov/trading-bot:latest .
   ```
3. Отправьте образ в Docker Hub:
   ```bash
   docker push timurarbuzov/trading-bot:latest
   ```

## ШАГ 4: Настройка нового сервера

1. Подключитесь к новому серверу по SSH (пароль из почты):
   ```bash
   ssh root@IP_ВАШЕГО_СЕРВЕРА
   ```
2. Установите Docker одной командой:
   ```bash
   curl -fsSL https://get.docker.com -o get-docker.sh && sh get-docker.sh
   ```
3. Создайте папку для бота и перейдите в нее:
   ```bash
   mkdir -p /root/trade_bot
   cd /root/trade_bot
   ```

---

## ШАГ 5: Создание файлов конфигурации на сервере

Находясь в папке `/root/trade_bot`, вам нужно создать три файла.

### 1. Файл docker-compose.yml
Выполните команду ниже (скопируйте и вставьте целиком в терминал сервера):
```bash
cat << 'EOF' > docker-compose.yml
services:
  trading-bot:
    image: timurarbuzov/trading-bot:latest
    container_name: trading-bot
    restart: unless-stopped
    env_file:
      - .env
    expose:
      - "8000"
    volumes:
      - ./config.json:/app/config.json
      - ./config.db:/app/config.db
      - ./bot.log:/app/bot.log
    logging:
      driver: "json-file"
      options:
        max-size: "10mb"
        max-file: "3"

  caddy:
    image: caddy:latest
    container_name: caddy
    restart: unless-stopped
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./Caddyfile:/etc/caddy/Caddyfile
      - caddy_data:/data
      - caddy_config:/config

volumes:
  caddy_data:
  caddy_config:
EOF
```

### 2. Файл Caddyfile (Настройка HTTPS)
Замените `ВАША_ПОЧТА@example.com` и `ВАШ_ДОМЕН.ru` на реальные данные, затем выполните команду:
```bash
cat << 'EOF' > Caddyfile
{
    email ВАША_ПОЧТА@example.com
}

ВАШ_ДОМЕН.ru {
    reverse_proxy trading-bot:8000
}
EOF
```

### 3. Файл .env (Секреты)
Откройте текстовый редактор:
```bash
nano .env
```
Вставьте туда ваши настройки. **ОБЯЗАТЕЛЬНО** укажите ваш домен в `WEBAPP_URL`:
```env
# Примеры настроек (замените на свои реальные ключи)

# Bybit
BYBIT_API_KEY=ваш_ключ
BYBIT_API_SECRET=ваш_секрет
BYBIT_TESTNET=true

# Telegram
TELEGRAM_BOT_TOKEN=токен_от_BotFather
TELEGRAM_CHAT_ID=ваш_id_в_телеграме
# Укажите HTTPS адрес вашего домена для работы кнопки WebApp
WEBAPP_URL=https://ВАШ_ДОМЕН.ru

# Безопасность Webhook
WEBHOOK_PASSPHRASE=супер_сложный_пароль_для_tradingview

# Остальные настройки
EXCHANGE_ID=bybit
USE_TESTNET=true
```
Сохраните файл: `Ctrl+O` -> `Enter` -> `Ctrl+X`.

## ШАГ 6: Запуск бота

Всё готово! Запускаем бота:

```bash
docker compose pull
docker compose up -d
```

### Проверка:
1. Введите `docker ps`. Вы должны увидеть два контейнера (`trading-bot` и `caddy`), которые запущены (Up).
2. Зайдите в браузер по адресу `https://ВАШ_ДОМЕН.ru/static/index.html`. Вы должны увидеть вашу панель управления, а в адресной строке должен появиться замочек (безопасное HTTPS соединение).
3. Напишите боту в Telegram команду `/start`. Нажмите на кнопку под сообщением — должна открыться WebApp.

---

## ШАГ 7: Настройка TradingView

В TradingView при создании алерта укажите:
- **Webhook URL:** `https://ВАШ_ДОМЕН.ru/webhook`
- **Message:** В JSON обязательно добавьте: `"passphrase": "ваш_пароль_из_env"`. Пример:
  ```json
  {
    "passphrase": "супер_сложный_пароль_для_tradingview",
    "action": "buy",
    "ticker": "BTCUSDT",
    "multiplier": 1,
    "leverage": 10
  }
  ```

Готово! Ваш бот задеплоен на современном сервере во Франкфурте, защищен HTTPS и готов к торговле.
